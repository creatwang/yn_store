**目标**: 搭建 monorepo 基础结构，确保所有应用可以正常启动。

### 任务清单

- [ ] 初始化 monorepo（pnpm workspace + Turborepo）
- [ ] 创建 `apps/admin`（Next.js 15 + Shadcn/ui）
- [ ] 创建 `apps/storefront`（Astro 5）
- [ ] 创建 `packages/db`（Drizzle ORM + 连接 Supabase）
- [ ] 创建 `packages/shared`（共享 TypeScript 类型）
- [ ] 配置 `turbo.json`、`tsconfig` references
- [ ] 验证：所有 app 能正常启动

### 交付物

```
apps/admin/src/app/layout.tsx          # Next.js 根布局
apps/admin/src/app/page.tsx            # Admin 首页（重定向到 dashboard）
apps/admin/src/app/(auth)/login/page.tsx  # 登录页占位
apps/storefront/src/pages/index.astro  # Storefront 首页
packages/db/src/index.ts               # Drizzle schema 导出入口
packages/db/src/connection.ts          # Supabase 数据库连接配置
packages/db/drizzle.config.ts          # Drizzle 配置文件
packages/shared/src/types/index.ts     # 共享类型定义
turbo.json                             # Turborepo 管道配置
pnpm-workspace.yaml                    # pnpm workspace 配置
```

### 技术要点

1. **pnpm-workspace.yaml** 需包含：
   ```yaml
   packages:
     - 'apps/*'
     - 'packages/*'
   ```

2. **turbo.json** 管道定义：
   - `build`：依赖 `^build`
   - `dev`：持久任务，无缓存
   - `lint`：依赖 `^build`

3. **packages/db** 连接配置需从环境变量读取 `DATABASE_URL`（Supabase 提供的 PostgreSQL 连接串）

4. **Next.js 15** 使用 App Router，开启 Turbopack

### 认证流程

```
用户输入邮箱密码
  → POST /auth/user/emailpass { email, password }
  → 返回 { token }
  → 存入 HttpOnly cookie
  → 后续请求头携带 Authorization: Bearer {token}
  → Token 过期前自动 refresh
```

## Phase 3: 购物车 + 结账 (Day 8-10)

**目标**: 实现完整的购物流程，从加入购物车到结账完成。

### 任务清单

- [ ] Storefront：购物车组件（React Island，`client:load`）
- [ ] Storefront：购物车页面（商品列表、数量调整、删除）
- [ ] Storefront：结账流程（地址 → 配送 → 支付 → 确认）
- [ ] Storefront：地址表单（收货地址 + 账单地址）
- [ ] Storefront：配送方式选择
- [ ] Storefront：支付集成

### 结账流程

```
第 1 步：地址信息
  → POST /store/carts/{id}  (shipping_address, billing_address, email)

第 2 步：配送方式
  → GET /store/shipping-options?cart_id={id}
  → POST /store/carts/{id}/shipping-methods  (option_id)

第 3 步：支付
  → POST /store/payment-collections/{id}/payment-sessions  (provider_id)
  → 前端处理支付（如 Stripe Elements）

第 4 步：确认下单
  → POST /store/carts/{id}/complete
  → 返回 order 对象
  → 跳转到订单确认页
```

### API 调用

| 功能 | 方法 | 端点 |
|------|------|------|
| 创建购物车 | POST | `/store/carts` |
| 获取购物车 | GET | `/store/carts/{id}` |
| 添加商品行 | POST | `/store/carts/{id}/line-items` |
| 更新商品行 | POST | `/store/carts/{id}/line-items/{line_id}` |
| 删除商品行 | DELETE | `/store/carts/{id}/line-items/{line_id}` |
| 更新购物车 | POST | `/store/carts/{id}` |
| 添加配送方式 | POST | `/store/carts/{id}/shipping-methods` |
| 完成结账 | POST | `/store/carts/{id}/complete` |

### 购物车状态管理

使用 React Context + localStorage 存储 `cart_id`：
- 首次访问时创建购物车（`POST /store/carts`）
- `cart_id` 存入 localStorage
- 页面加载时通过 `cart_id` 恢复购物车状态
- 结账完成后清除 `cart_id`，下次访问创建新购物车

| 功能 | 方法 | 端点 |
|------|------|------|
| 客户列表 | GET | `/admin/customers` |

| 更新库存 | POST | `/admin/inventory-items/{id}/location-levels/{location_id}` |
| 仓库列表 | GET | `/admin/stock-locations` |

### 库存管理要点

- **库存项 (Inventory Item)**: 对应商品变体的库存信息
- **库存级别 (Inventory Level)**: 某个库存项在某个仓库的具体数量
  - `stocked_quantity`: 实际库存数
  - `reserved_quantity`: 已预留数（已下单未发货）
  - 可用库存 = stocked - reserved
- **预留项 (Reservation Item)**: 订单创建时预留库存

  type: 'standard' | 'buyget';    // 类型
  is_automatic: boolean;           // 是否自动应用
  campaign_id?: string;            // 关联活动
