# Medusa v2.15.3 API 端点参考

> ⚠️ **参考文档**（Medusa 官方 ~220 路由）。**本仓库实际挂载** → [PROJECT_STATUS.md](./PROJECT_STATUS.md) · [API-IMPLEMENTATION-GAP.md](./API-IMPLEMENTATION-GAP.md)

> 本文档列出 Medusa v2.15.3 所有 REST API 端点，基于 `@medusajs/medusa/dist/api/` 文件系统路由提取。  
> **本仓库实际实现与缺口** → [API-IMPLEMENTATION-GAP.md](./API-IMPLEMENTATION-GAP.md)（勿用本文档行数推算完成度）。

---

## 认证说明

| 项目 | 说明 |
|------|------|
| **Admin API** | 需要 JWT Bearer Token（通过 `POST /auth/user/emailpass` 获取），请求头：`Authorization: Bearer <token>` |
| **Store API** | 需要 Publishable API Key，请求头：`x-publishable-api-key: <key>` |
| **分页** | `offset` / `limit` 参数，默认 `limit=20` |
| **排序** | `order` 参数，如 `order=created_at` 或 `order=-created_at`（降序） |
| **过滤** | `fields` 参数选择返回字段，`expand` 参数展开关联关系 |

## Auth API 端点

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/auth/:actor_type/:provider` | 认证（如 `POST /auth/user/emailpass`） |
| POST | `/auth/:actor_type/:provider/callback` | OAuth 回调 |
| POST | `/auth/:actor_type/:provider/register` | 注册 |
| POST | `/auth/token/refresh` | 刷新 Token |

**actor_type 常用值：**

| actor_type | 说明 |
|-----------|------|
| `user` | 管理员用户 |
| `customer` | 店铺客户 |

**provider 常用值：**

| provider | 说明 |
|----------|------|
| `emailpass` | 邮箱密码认证 |
| `google` | Google OAuth |
| `github` | GitHub OAuth |

**认证流程示例：**

```bash
# 1. 管理员登录获取 Token
curl -X POST http://localhost:7000/auth/user/emailpass \
  -H "Content-Type: application/json" \
  -d '{"email": "admin@example.com", "password": "supersecret"}'

# 返回: { "token": "eyJhbGciOiJIUzI1NiIs..." }

# 2. 使用 Token 调用 Admin API
curl http://localhost:7000/admin/products \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..."

# 3. 客户登录
curl -X POST http://localhost:7000/auth/customer/emailpass \
  -H "Content-Type: application/json" \
  -d '{"email": "customer@example.com", "password": "password123"}'

# 4. 使用 Token + Publishable API Key 调用 Store API
curl http://localhost:7000/store/products \
  -H "Authorization: Bearer <customer_token>" \
  -H "x-publishable-api-key: <your_publishable_key>"
```
