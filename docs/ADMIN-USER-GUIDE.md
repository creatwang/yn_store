## 创建产品：「管理库存」与「库存套件」

### 三个概念

| 名称 | 是什么 |
|------|--------|
| **变体（Variant）** | 顾客下单购买的那一行，例如「礼盒 / 默认」 |
| **库存项（Inventory Item）** | 仓库里真正记「还有多少件」的记录（可有 SKU、多仓数量） |
| **变体 ↔ 库存项** | 规则：卖 1 个该变体时，从仓库扣哪些库存、各扣多少 |

### 「管理库存」——总开关

- **勾选**：系统跟踪并扣减库存；下单/退货会改数量；可在「库存」菜单查看。
- **不勾选**：不跟踪库存。适合数字商品、服务、或库存完全在外部系统管理。

提示文案含义：**启用后，我们会在创建订单和退货时为您更改库存数量。**

### 「库存套件」——子模式（仅当已管理库存）

- **不勾选（最常见）**：**1 个变体 → 1 个库存项**。保存产品时系统通常会自动建立关联。
- **勾选**：**1 个变体 → 多个库存项 + 各自数量**（套装/BOM）。例如「礼盒」= 酒×2 + 饼×1 + 包装×1。需在创建流程第 4 步 **「库存套件」** 页手动选择库存项与数量。

提示文案含义：**此变体是否由多个库存项目组成？**

### 两者关系（必记）

1. **必须先勾选「管理库存」，「库存套件」才能勾选**（未管理库存时该列为灰色）。
2. **大多数实物商品**：只勾 **管理库存**，**不要**勾库存套件。

| 管理库存 | 库存套件 | 效果 |
|:--------:|:--------:|------|
| 否 | 否 | 不跟踪库存 |
| 是 | 否 | 普通单品，1 对 1 库存 |
| 是 | 是 | 组合商品，需配置多库存项 |

### 与「允许缺货订购」的区别

- **允许缺货订购**：在已管理库存的前提下，库存为 0 时是否仍允许顾客下单（超卖策略）。
- 与是否套件无关。

### 创建流程对应步骤

1. **详情 / 组织** — 标题、分类等  
2. **变体** — 勾选管理库存、库存套件，填写价格（需先在 **设置 → 商店 → 货币** 添加支持货币，否则无价格列）  
3. **库存套件**（可选）— 仅当至少一个变体同时开启「管理库存」且「库存套件」时出现  

### 示例

- **T 恤 L 码**：管理库存 ✅，库存套件 ❌  
- **春节礼盒（含酒×2、饼×1）**：管理库存 ✅，库存套件 ✅，并在第 4 步配置各组件数量

## 设置：商店与货币

### 编辑商店时「默认货币」下拉为空？

编辑商店表单的**默认货币**选项来自当前商店已配置的 **支持货币**（`supported_currencies`），不是全球货币主数据列表。

1. 打开 **设置 → 通用 → 商店**。
2. 在 **货币** 区域点击 **添加货币**，勾选需要的货币（如 USD、CNY）并保存。
3. 再打开 **编辑商店**，即可在「默认货币」中选择，并指定哪一种是默认。

若 **默认地区**、**默认位置** 也为空：需先在 **设置 → 地区** 创建地区，在 **设置 → 位置** 创建库存位置；列表为空时对应下拉会禁用。

## 设置：地区与支付供应商

### 地区详情里「国家」列表为空？

Medusa v2 把国家存在 **`region_country`** 表（按 `region_id` 关联），不是旧的 `country` 表。若自定义后端只查了不存在的 `country` 表，即使用官方种子库也会在页面上看不到德国、法国等。

修复后 `GET /admin/regions/:id` 会返回 `countries: [{ iso_2, display_name, ... }]`，与官方 Admin 一致。

### 编辑/创建地区时「支付供应商」无选项？

支付供应商由**后端支付模块**注册并写入 `payment_provider` 表，Admin 中**没有**单独的「新建支付供应商」表单。

- 下拉列表来自 **`payment_provider` 表**里 `is_enabled = true` 的行；与官方共用同一库时，若表里只有 `pp_system_default`，下拉也只会有一项（可用 SQL 查表核对）。
- 本仓库启动时会 `INSERT` 默认的 **`pp_system_default`**（若尚不存在）。
- 接入 Stripe 等插件后，官方会在表中增加如 `pp_stripe_stripe`；地区创建/编辑里勾选的是「该地区启用哪些已注册的供应商」。

配置入口在**服务端**（环境变量、支付模块、`apps/server/src/lib/providers/`），不是 Dashboard CRUD。

## 设置：税收地区

- 国家/地区级税收区域详情页会列出下级省/州（若该国支持）。
- 若打开详情白屏，请确认后端已返回 `children` 数组（见 `settings.service.ts` 税收地区详情接口）。
